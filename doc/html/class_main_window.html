◆ updateSelectCoord()

void MainWindow::updateSelectCoord ( )

This function uses the stored values of iMin, jMin, iMax, and jMax to update the displayed coordinates. Each of these stored values is zero - indexed, so the displayed value is incremented by 1.

This function also (1) calls DisplayGrid::clearHighlight() to remove all possible highlighting in the grid, (2) sets the show/hide button to "Show", and (3) calls MainWindow::enableGenerateButtons() with parameter false to disable all of the generating buttons

◆ initDispSizer()

void MainWindow::initDispSizer ( )
private

Method

First, check to see if the Knot has been initialized already. If it has not, then neither has the DisplayGrid or its sizer, so the sizer is initialized with 2 stretch spacers. It the Knot has been initialized, then delete the Knot and Destroy() the DisplayGrid.

Regardless of the above, initialize the Knot with the member variables h and w and the status bar. Then initialize the DisplayGrid with the newly generated Knot, and insert it between the stretch spacers in its sizer.

◆ gridRegenFunction()

void MainWindow::gridRegenFunction ( wxCommandEvent &  evt)
private

Method

First, check if the values in the boxes are numbers. If not, send a mxMessageBox with an error message and return. Then check if the numbers in the boxes are positive integers. If not, send a different error message and return.

Next, update h and w member variables, then reinitialize the DisplayGrid using MainWindow::initDispSizer().

Then, reset the select coordinates with MainWindow::resetSelectCoord(), regenerate and export textbox with MainWindow::regenExportBox(), and reset the knot wrapping wxMenuItem objects.

Lastly, refresh the minimum size of the window.

◆ selectToggleFunction()

void MainWindow::selectToggleFunction ( wxCommandEvent &  evt)
private

Method

If the text in the show/hide button is "Show", then first fix the selection coordinates with MainWindow::fixSelectCoords(), call DisplayGrid::highlightSelection() with the index member variables, set the button text to be "Hide", and call MainWindow::enableGenerateButtons() to enable the buttons.

If the text in the show/hide button is otherwise (which can only be "Hide"), then first call DisplayGrid::clearHighlight(), set the button text to be "Show", and call MainWindow::enableGenerateButtons() with paramater false to disable all buttons.

◆ enableGenerateButtons()

void MainWindow::enableGenerateButtons ( bool  enable = true)
private

This function allows the Knot generate functions to have no conditional operation, and just assume that the parameters are valid. Instead of checking for symmetry conditions within the generating function itself, the user is disallowed from pressing the button.

Parameters
enableTells the function whether to enable (conditionally) or disable (fully) the generating buttons, has a default value of true

Method

To conditionally enable the buttons, first call the Knot::check____Sym() functions on the current selection and store the outputs. Then enable each of the buttons is the proper symmetry condition has been met.

If enable is false, then disable each of the buttons.

◆ generateKnot()

void MainWindow::generateKnot ( wxCommandEvent &  evt)
private

The Knot::generate() function uses the status bar, so first store the current displayed message.

Each of the generating buttons has its symmetry type as its ID value. Get this symmetry from the event call. Then, forward on this symmetry to Knot::generate() with the selection coordinates to be generated. If this function returns true, then the Knot has been generated successfully, so update the DisplayGrid with DisplayGrid::drawKnot() and update the export text box with MainWindow::showExportBox(). If the generate function returns false, then display an error message as a wxMessageBox.

At the end, set the status bar back to the message which was displayed at the beginning of the function, and re-enable the generate buttons.
