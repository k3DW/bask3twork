Knot::tryGenerating ( std::optional< GlyphVec2 > &  glyphGrid,
const int  iMin,
const int  jMin,
const int  iMax,
const int  jMax,
const Side  ignoreSides = Side::NONE,
const GlyphFlag  boolFlags = GlyphFlag::NONE 
) const
private

Only generates from row iMin to row iMax, and from column jMin to column jMax. This function assumes iMin <= iMax and jMin <= jMax, and does not check the values.

The ignoreSides flag only affects the side-most Glyphs. For example, if ignoreSides includes Side::LEFT, then the left side connection will only be ignored on the leftmost Glyphs of the selection.

The boolFlags flag affects all Glyphs in the selection. All flags which are not specified are ignored, instead of being checked for a value of zero.

Parameters
glyphGridThe 2D std::vector of Glyph pointers to pass into the function to be edited, as an optional
iMinThe zero-indexed upper row of the selection visually (lower numerically)
iMaxThe zero-indexed lower row of the selection visually (higher numerically)
jMinThe zero-indexed leftmost column of the selection
jMaxThe zero-indexed rightmost column of the selection
ignoreSidesThe sides to ignore, as an or-composition of Side values
boolFlagsThe conditions which need to be met, as an or-composition of GlyphFlag values

Methodology

First, make a copy of glyphGrid to make changes without affecting the input.

Then, set the const Glyph* pointers of the copy to nullptr, if they are within the selection. This signifies that these values are not set, since they are being regenerated in this function.

Next, generate the glyphs by iterating over i,j coordinates within the selection. For each i,j pair, do the following 3 steps.

(1) Generate the vector of all possible glyphs that can fit into this spot, using the PossibleGlyphs() function, passing on the boolFlags parameter directly. For each Connection parameter, there are 4 possible cases, in order: (a) The side should be ignored based on ignoreSide, therefore the parameter should be Connection::DO_NOT_CARE. (b) This glyph is on the outside for this particular side, which is treated as Connection::EMPTY. (c) The next glyph on this particular side is not yet assigned, therefore the parameter should be Connection::DO_NOT_CARE. (d) The general case, where none of the other special cases apply, therefore the parameter should be the connection on the opposite side, from the neighbouring glyph on this particular side.

(2) If there are no possible glyphs for this space, set glyphGrid to the null optional and end the function.

(3) If step 2 did not end the function, set the glyph in the copy as a random choice from the generated possibilities.

At the end of the function, set glyphGrid equal to the created copy.
